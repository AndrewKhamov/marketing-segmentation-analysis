## 📌 Описание проекта

Проект посвящён анализу клиентской базы для CRM-маркетинга.

Изначальная задача была решена в SQL: для каждого клиента был рассчитан сегмент на основе количества доставленных заказов и давности регистрации.

В ноутбуке SQL-выгрузка используется как аналитический датасет для следующего шага - оценки структуры клиентской базы, покупательской активности и поиска гипотез для персонализированных коммуникаций.

---

## 🎯 Цель анализа

Понять, как распределяются клиенты по сегментам, какие группы формируют основную активность и где есть потенциал для роста повторных покупок.

### 🔍 Ключевые вопросы

- Какая структура клиентской базы по сегментам?
- Какие сегменты формируют основную покупательскую активность?
- Есть ли потенциал для реактивации клиентов без заказов?
- Отличается ли поведение клиентов по городам?
- Может ли опыт доставки влиять на повторные покупки?
- Насколько стабилен приток новых регистраций?

---

## 📋 Исходная задача

Необходимо провести сегментацию клиентов по состоянию на **31 марта 2024 года**.

Для каждого клиента рассчитывались следующие поля:

| Поле | Описание |
|---|---|
| `customer_id` | идентификатор клиента |
| `customer_city` | город клиента |
| `registration_date` | дата регистрации |
| `days_since_registration` | количество дней с даты регистрации |
| `orders_count` | число доставленных заказов |
| `purchase_events_count` | число событий покупки |
| `avg_delivery_days` | среднее время доставки в днях |
| `segment` | клиентский сегмент |

---

## 🧩 Правила сегментации

| Сегмент | Критерий | Стратегия коммуникации |
|---|---|---|
| **Постоянный** | 3 и более доставленных заказа | Эксклюзивные предложения и программа лояльности |
| **Разовый** | 1-2 доставленных заказа | Напоминание о магазине и стимул к повторной покупке |
| **Неактивный** | 0 заказов, регистрация более 30 дней назад | Реактивационные акции |
| **Новый** | 0 заказов, регистрация 30 дней назад или меньше | Приветствие и помощь с первой покупкой |

## 🚀 Ход анализа

### 1. Подготовка данных
- проверка типов данных;
- обработка пропусков и дублей;
- приведение дат и числовых полей к корректным форматам;
- проверка категориальных признаков.

### 2. Анализ клиентских сегментов
- распределение клиентов по сегментам;
- оценка количества заказов по сегментам;
- анализ клиентов без доставленных заказов;
- сравнение плановых и фактических показателей.

### 3. Географический анализ
- сравнение структуры сегментов по городам;
- поиск городов с повышенной долей недоставленных заказов;
- определение потенциальных зон для оптимизации.

### 4. Формирование выводов
- оценка качества клиентской базы;
- поиск проблемных зон в воронке;
- подготовка рекомендаций для CRM-коммуникаций.

---

## 📈 Основные выводы

- Клиентская база сильно смещена в сторону пользователей без доставленных заказов: основную часть составляют сегменты **Неактивных** и **Новых** клиентов.

- Основную массу доставленных заказов формируют **Разовые** клиенты. Сегмент **Постоянных** клиентов достатчно небольшой, но имеет высокую ценность.

- В сегментах **Неактивных** и **Разовых** заметна повышенная доля клиентов с недоставленными заказами. Это может указывать на проблемы между purchase-событием и успешной доставкой.

- Были выявлены города с повышенной долей недоставленных заказов, что требует дополнительного анализа логистики или клиентского пути.

- Приток новых регистраций остаётся стабильным, поэтому ключевая задача - не только привлечение новых пользователей, но и повышение конверсии в первый и повторный заказ.

- География клиентов может использоваться для приоритизации маркетинговых кампаний и настройки коммуникаций по городам, поскольку города заметно отличаются по доле постоянных, разовых и неактивных клиентов.

---

## 💡 Рекомендации

- Запустить реактивационные кампании для **Неактивных** клиентов, особенно для тех, у кого были purchase-события, но не было доставленных заказов.

- Подготовить персональные предложения для клиентов с **1-2 заказами**, чтобы стимулировать переход в сегмент постоянных покупателей.

- Проанализировать этапы между покупкой и доставкой, чтобы найти причины недоставленных заказов.

- Отдельно мониторить города с высокой долей незавершённых заказов.
